## Routing Packets
### Default Gateway
- End hosts like PC1 and PC4 can send packets directly to destinations in their connected network
	- PC1 is connected to 192.168.1.0/24, PC4 is connected to 192.168.4.0/24
- To send packets to destinations outside of their local network, they must send the packets to their **Default Gateway**.
- The **default gateway** configuration is also called a **default route**.
	- It is a route to 0.0.0.0/0 = all netmask bits set to 0. Includes all addresses from 0.0.0.0 to 255.255.255.255
	- the **default route** is the *least specific route* possible, because it includes <u>all</u> IP addresses. 0.0.0.0/0 = 4,294,967,296 IP addresses
- End hosts usually have no need for any more specific routes
	- they just need to know: to send packets outside of my local network, i should send them to my default gateway.
### Static Routes
##### Let's send a packet to PC4 from PC1 in [[staticrouting-topology_example.png]]
- When R1 receives the frame from PC1, it will de-encapsulate it (remove L2 header/trailer) and look at the nested packet.
- It will check the routing table from the most-specific matching route
- R1 has no matching routes in its routing table
- To properly forward the packet, R1 needs a route to the destination network (192.168.4.0/24)
	- Routes are instructions: *To send a packet to destinations in network 192.168.4.0/24, forward the packet to next hop Y.*
### Static Route Configuration
- Each router in the path needs <u>two</u> routes: a route to 192.168.1.0/24 and a route to 192.168.4.0/24
	- This ensures **two-way reachability** (i.e. PC1 can send packets to PC4, PC4 can send packets to PC1)
		- Routers don't need routes to all networks in the path to the destination.
			- ex) R1 doesn't need a route to 192.168.34.0/24
- The following image demonstrates how next-hop (static routes) are chosen using the example topology.
		[[staticrouting-static_route_configuration.png]]
##### R1(config)# ip route *ip-address netmask next-hop*
- Where,
	- *ip-address* is the intended destination
	- *netmask* is the netmask of said destination
	- *next-hop* is the closest interface available to reach said destination
		``R1(config)# ip route 192.168.4.0 255.255.255.0 192.168.13.3``
		- This command will add a static route to 192.168.4.0/24 to R1's routing table via interface 192.168.13.3
		- The new entry:
				``S     192.168.4.0/24 [1/0] via 192.168.13.3``
			- The \[1/0] displayed in static routes means \[**Administrative Distance/Metric**]. This concept is covered later in the course supposedly lol
R3(config)#ip route 192.168.1.0/24 255.255.255.0 192.168.13.1
R3(config)#ip route 192.168.4.0/24 255.255.255.0 192.168.34.4
R4(config)#ip route 192.168.1.0/24 255.255.255.0 192.168.34.3
### Static Route Configuration with *exit-interface*
##### R2(config)# ip route *ip-address nextmask exit-interface*
- This allows you to set an interface name (i.e. *g0/0*) instead of a next-hop as the route for intended destinations
- If only *exit-interface* is used, it relies on a feature called **Proxy ARP** to function. It's usually not a problem but generally you can stick to *next-hop* or *exit-interface next-hop*.
##### R2(config)# ip route *ip-address nextmask exit-interface*
- You can also just do both lol
### Default Route
- A **default route** is a route to 0.0.0.0/0
	- **least specific route possible**
- If the router doesn't have any more specific routes that match a packet's destination IP address, the router will forward the packet using the **default route**.
- A default route is often used to direct traffic to the Internet.
	- More specific routes are used for destinations in the internal corporate network
	- Traffic to destinations outside of the internal network is sent to the Internet
```R1
R1# show ip route
[codes]
Gateway of last resort is not set
```
- This means *no **default route** has been configured yet.*
- Set it with:
##### R1(config)# ip route 0.0.0.0 0.0.0.0 *internet-providing-interface-ip*
- For [[staticrouting-default_route_example.png]], this is the command to set it:
	``ip route 0.0.0.0 0.0.0.0 203.0.113.2``
- In R1's routing table, the new entry appears as:
	``S*     0.0.0.0 [1/0] via 203.0.113.2``
	- The S* indicates *candidate default*, meaning candidate to become the default route.
